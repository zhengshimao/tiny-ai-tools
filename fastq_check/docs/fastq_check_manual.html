fastq_check 使用说明

一个 Rust 编写的 FASTQ/FASTQ.GZ 格式检查工具，支持单端、双端、平台推测、质量字符范围统计、JSON 报告和可选 pigz 多线程解压。

本文档为自包含 UTF-8 HTML 文件，示例基于 example 目录下的非压缩 FASTQ 文件。

1.1 功能摘要

FASTQ 格式检查

检查 header、plus 行、序列与质量长度是否一致、截断和 gzip EOF 等问题。

单端/双端支持

仅提供 --read1 时执行单端检查；同时提供 --read2 时执行双端配对一致性检查。

并行处理

读取线程按 batch 发送数据，Rayon worker 并行校验，线程局部统计后 reduce。

平台推测

可启用 --infer-platform，从 header 推测 Illumina、BGI/MGI、PacBio、Oxford Nanopore 或 Unknown。

3. 输入文件格式

3.1 FASTQ 四行结构

FASTQ record
@header
ACGTN...
+
quality_ascii...

脚本按四行一组读取记录。header 必须以 @ 开始，第三行必须以 + 开始，sequence 与 quality 字符数必须一致。

5. 参数说明

5.1 参数表

参数默认值作用注意事项
--read1 / -1必填输入 R1 或单端 FASTQ/FASTQ.GZ单端模式只需提供该参数
--read2 / -2无输入 R2 FASTQ/FASTQ.GZ提供后启用双端配对检查
--threads0Rayon worker 线程数0 表示由 Rayon 自动选择
--batch-size8192每个 batch 包含的 FASTQ records 数不是文件数；8192 records 等于 32768 行
--pigzfalse对 .gz 文件使用外部 pigz 解压不指定则保持 flate2 读取
--pigz-threads8传给 pigz 的线程数必须大于 0
--pigz-pathpigzpigz 可执行文件路径PATH 不生效时可指定完整路径
--strict-basesfalse限制碱基为 A/C/G/T/N小写或 IUPAC 扩展碱基会报错
--detect-qualityfalse统计 quality ASCII 范围并推测编码min/max 字段是 ASCII，不是解码后的 Q 值
--infer-platformfalse从 header 推测平台结果为启发式，不作为强校验
--platform-sample-size10000每个文件用于平台推测的 reads 数只影响平台推测；完整性检查仍扫描全文件
--json-report无输出 pretty JSON 报告不指定则只输出终端摘要

5.2 pigz 解压

pigz example
target/release/fastq_check ^
  --read1 input_R1.fq.gz ^
  --read2 input_R2.fq.gz ^
  --pigz ^
  --pigz-path /path/to/pigz ^
  --pigz-threads 8 ^
  --threads 8 ^
  --json-report report.json
pigz 只负责 gzip 解压；FASTQ 检查和统计仍由 Rust 程序执行。非压缩 FASTQ 即使指定 --pigz 也会按普通文本读取。

8. 使用注意事项

8.1 quality 字段说明

min_qual_ascii 和 max_qual_ascii 是 FASTQ 第 4 行 quality 字符的 ASCII 编码范围，不是已经解码后的 Phred Q 值。

例如字符 I 的 ASCII 是 73。如果按 Phred+33 解码，则 Q=40；如果按 Phred+64 解码，则 Q=9。因此脚本先报告 ASCII 范围，再给出启发式 phred_detected。

8.2 平台推测说明

--infer-platform 使用 header 模式做启发式推测。输出可能为 Illumina、BGI/MGI、PacBio、Oxford Nanopore 或 Unknown。

--platform-sample-size 只限制用于平台推测的 reads 数，FASTQ 完整性检查仍会扫描完整文件。

平台推测不应作为强制格式校验。公共数据库下载的 FASTQ 可能经过转换，header 不一定保留原始测序仪格式。
