run_download_zenodo.py 使用说明

该脚本用于读取 Zenodo record 元数据，自动生成 wget、axel、aria2c 三套下载脚本，并提取 md5 校验文件、URL 列表和 manifest 表格。适合批量下载 Zenodo 数据集文件， 尤其适合包含多个大文件的生信数据记录。

1. 脚本概述

核心用途

从 Zenodo API 获取文件列表，生成可直接执行的下载脚本。

下载工具

支持 wget、axel、aria2c 三种命令格式。

校验支持

自动提取 Zenodo 元数据中的 md5: 校验值，输出为 md5sum -c 可用格式。

当前脚本文件为 run_download_zenodo.py，仅使用 Python 标准库，不依赖第三方 Python 包。 生成的下载脚本面向 Linux/WSL/bash 环境执行。

2. 运行环境

  • Python 3.6 或更高版本。
  • 可访问 https://zenodo.org/api/records/<record_id> 的网络环境。
  • 执行下载脚本时，需要安装对应下载工具: wget、axel 或 aria2c。
  • Windows 工作区建议在 WSL 中执行生成后的 .sh 下载脚本。

4. --record 写法

--record 是必填参数，支持多种输入形式。脚本会统一解析为 Zenodo record ID。

输入类型 示例 解析结果
纯数字 ID 15596052 15596052
Zenodo 页面链接 https://zenodo.org/records/15596052 15596052
Zenodo API 链接 https://zenodo.org/api/records/15596052 15596052
DOI 链接 https://doi.org/10.5281/zenodo.15596052 15596052
DOI 字符串 10.5281/zenodo.15596052 15596052
带 doi: 前缀 doi:10.5281/zenodo.15596052 15596052

5. 脚本思路

解析参数 读取 --record、--outdir、--prefix、--timeout
识别 record ID 兼容 ID、Zenodo URL、DOI URL、DOI 字符串
请求 Zenodo API 使用 urllib 下载 JSON，网络错误自动重试
提取文件字段 读取 key、size、checksum、links.self
写出结果 生成 3 个下载脚本、MD5、URL、manifest

脚本不会直接下载数据文件，只生成下载命令。这样可以先检查 manifest、选择合适工具，再在 WSL/Linux 中执行对应脚本。

7. 函数功能

函数名 输入 输出 功能说明
check_output_dir(path) 输出目录路径 None 创建并校验输出目录是否为可写目录。
parse_record_id(record) ID、URL 或 DOI 字符串 record ID 字符串 从多种 Zenodo 记录写法中提取数字 ID。
fetch_json(url, timeout, retries, retry_delay) API URL、超时秒数、重试参数 解析后的 JSON 对象 下载 Zenodo API JSON；对网络错误、HTTP 429、5xx 执行重试。
extract_files(record_data) Zenodo record JSON 文件信息列表 提取文件名、下载链接、大小和 MD5。
sh_quote(text) 任意字符串 shell 安全单引号字符串 防止文件名中的空格或单引号破坏 bash 命令。
format_size(size) 字节数 易读大小字符串 将字节数格式化为 B、KiB、MiB、GiB 或 TiB。
write_lines(path, lines) 输出路径和文本行 None 用 UTF-8 写出脚本、MD5 或 URL 列表。
write_manifest(path, files) manifest 路径和文件信息列表 None 写出包含 filename、size、size_bytes、md5、url 的 TSV。
build_wget_script(files) 文件信息列表 脚本文本行列表 生成 wget -c 下载命令。
build_axel_script(files) 文件信息列表 脚本文本行列表 生成 axel --insecure -c -n 20 下载命令。
build_aria2c_script(files) 文件信息列表 脚本文本行列表 生成 aria2c -c -j 16 -x 16 -s 1 下载命令。
write_scripts(files, outdir, prefix) 文件信息、输出目录、文件名前缀 输出路径字典 统一写出所有下载脚本和辅助文件。

8. 输入说明

命令行参数

参数 是否必填 默认值 说明 示例
-r, --record 是 无 Zenodo record URL、API URL、数字 ID、DOI URL 或 DOI 字符串。 15596052
-o, --outdir 否 . 生成文件输出目录。目录不存在时自动创建。 scripts
-p, --prefix 否 zenodo_<record_id> 输出文件名前缀。 hiv_rna
-t, --timeout 否 60 单次 Zenodo API 请求超时时间，单位秒。 120

文件 格式 用途

12. 使用注意事项

  • 脚本只生成下载脚本，不会直接下载 Zenodo 数据文件。
  • 生成的 .sh 文件建议在 WSL/Linux/bash 环境中执行。
  • 下载大文件前应确认磁盘空间充足。示例记录总大小约 31.24 GiB。
  • wget、axel、aria2c 三套脚本任选其一执行即可，不需要全部执行。
  • 下载完成后建议运行 md5sum -c scripts/zenodo_15596052_md5.txt 校验文件完整性。
  • 如果 Zenodo API 短暂不可用，脚本会自动重试；非 429 的 4xx 错误通常表示 record 参数或权限存在问题。

13. 常见问题

提示 Cannot parse a Zenodo record ID

检查 --record 是否为支持的 ID、URL 或 DOI 格式。推荐先用纯数字 ID 测试。

下载脚本无法执行

确认当前环境是 bash，并且对应下载工具已经安装。Windows PowerShell 不能直接执行 bash 脚本，建议使用 WSL。

MD5 校验失败

通常表示文件未完整下载或内容损坏。删除失败文件或重新执行下载脚本，让下载工具续传或重新下载。

中文注释在 PowerShell 中显示乱码

这通常是控制台编码显示问题。脚本文件使用 UTF-8 编码保存，Python 按 UTF-8 读取正常。
