run_get_gsa_infor.py 使用说明

该脚本用于下载 GSA/GSA-human 项目的 metadata Excel、页面 HTML 和 md5sum.txt， 并基于 md5sum 生成 MD5 校验文件以及 aspera、axel、aria2c 三类下载辅助脚本。

脚本文件 run_get_gsa_infor.py
语言版本 Python 3.6+
外部依赖 无，仅标准库
当前支持 HRA, CRA, http URL

快速开始

最常见的用法是传入一个项目编号，脚本会在当前目录下创建 gsa_info_sample_项目编号 目录。

python run_get_gsa_infor.py -p HRA000425
python run_get_gsa_infor.py -p CRA000112
python run_get_gsa_infor.py -p https://download.cncb.ac.cn/gsa/CRA000112/

指定输出目录并强制刷新缓存：

python run_get_gsa_infor.py -p HRA000425 -o ./gsa_info_sample_HRA000425 --force

查看帮助信息：

python run_get_gsa_infor.py -h

命令行参数

参数 是否必需 默认值 说明 示例
-p, --project 必需 无 项目编号或完整下载 URL。当前支持 HRA、CRA 或 http 开头的输入。 HRA000425, CRA000112
-o, --outdir 可选 gsa_info_sample_<project_basename> 输出目录。不存在时自动创建。 -o ./result_HRA000425
--force 可选 False 强制重新下载已缓存的 HTML、Excel、md5sum.txt，并重新解析 URL。 --force

脚本思路与流程图

脚本主流程由 run_workflow() 组织。它先做输入校验和输出目录准备，再按项目类型下载 metadata，并解析最终数据下载 URL，最后基于 md5sum.txt 生成校验文件和下载脚本。

1 解析参数 读取 project、outdir、force，并记录日志。
2 校验项目 仅允许 HRA、CRA 或 http URL。
3 下载 metadata CRA 使用 GSA 接口，HRA 使用 GSA-human 接口。
4 获取下载 URL 优先读 url.txt；无缓存时解析或拼接 URL。
5 生成结果 下载 md5sum.txt，输出 MD5 和下载脚本。

项目类型分支

输入类型 metadata Excel index.html 最终下载 URL 备注
CRA000112 POST 到 /gsa/file/exportExcelFile /gsa/browse/CRA000112 从 HTML 中提取唯一 download URL 适用于普通 GSA 项目。
HRA000425 先从 HTML 提取 study_id，再下载 Excel /gsa-human/browse/HRA000425 https://download.cncb.ac.cn/gsa-human/HRA000425 检测到 Controlled access 时正常退出。
https://... 不下载 metadata Excel 不下载 browse HTML 直接使用输入 URL 适合用户已知下载根目录的情况。

函数功能说明

函数 输入 输出 功能说明 关键注意点
parse_args() 命令行参数 argparse.Namespace 解析 -p、-o、--force。 日志会记录参数字典。
validate_project(project) 项目编号或 URL None 检查输入是否以 HRA、CRA 或 http 开头。 不支持 XDA/XDB，会直接报错。
download_html(project, workdir, force) 项目编号、输出目录、刷新标记 index.html 路径 下载 CRA 或 HRA 的 browse 页面。 HRA 受控访问项目会输出 warning 并退出。
download_meta_gsa(project, workdir, force) CRA 编号 xlsx 文件 通过 GSA 接口 POST 下载 CRA metadata Excel。 仅接受 CRA[0-9]{6}。
download_meta_gsa_human(project, workdir, force) HRA 编号 xlsx 文件 下载 HRA browse 页面，提取 study_id 后下载 Excel。 依赖 HTML 中存在 study_id。
get_download_url(project, workdir, force) 项目编号或 URL 最终下载根 URL 读取或生成 url.txt 缓存。 CRA 从 HTML 提取，HRA 直接拼接，http 直接使用。
parse_md5sum(md5sum_file, project_base) md5sum.txt、项目名 列表 [(md5, path), ...] 过滤两列格式的 md5 行，并裁剪项目名前缀。 非两列行会被跳过。
write_md5_file(entries, output_file) md5 条目、输出文件 MD5.txt 或 MD5_all.txt 写出 md5\tbasename 格式。 第二列只保留文件名，不保留路径。
write_download_scripts(entries, url, workdir, prefix) md5 条目、URL、目录、前缀 三类下载脚本 生成 aspera、axel、aria2c 下载脚本。 Aspera key 不存在时会记录错误提示。
run_workflow(args) 参数对象 输出目录中的结果文件 主流程调度函数。 大多数异常在 main() 中被捕获并返回 1。

输入输出文件格式

输入

输入来源 格式 是否由用户提供 说明
项目编号 HRA000425 或 CRA000112 是 通过 -p/--project 传入。
完整下载 URL https://download.cncb.ac.cn/gsa/CRA000112/ 是 传入 URL 时跳过 metadata Excel 和 browse HTML 下载。

输出

文件名 格式 生成条件 用途
<project>.xlsx Excel 输入为 CRA 或 HRA 项目编号 项目 metadata 信息表。
index.html HTML 输入为 CRA 或 HRA 项目编号 browse 页面缓存。
url.txt 纯文本，一行 URL 总是生成 缓存最终下载根 URL。
md5sum.txt 两列文本 总是下载 远程原始 md5 文件。
MD5_all.txt md5\tfilename 总是生成 全部文件的校验表，可用于 md5sum -c。
MD5.txt md5\tfilename 存在 q.gz 结尾文件时 FASTQ 文件校验表。
run_download_all_files_using_*.sh Shell script 总是生成 全部文件下载脚本。
run_download_fastq_using_*.sh Shell script 存在 FASTQ 文件时 FASTQ 文件下载脚本。

使用注意事项

支持范围：当前脚本只支持 HRA、CRA 和完整 http URL。 XDA、XDB 等编号不会通过入口校验。
缓存策略：如果输出目录里已有非空的 index.html、.xlsx、url.txt 或 md5sum.txt， 默认会复用缓存。需要重新下载时加 --force。
Aspera key：生成 aspera 脚本时会检查 ~/.aspera/connect/etc/aspera01.openssh。 如果 key 不存在，脚本会打印下载提示，但仍会生成下载脚本。
受控访问：HRA 项目页面中如果包含 Controlled access，脚本会提示 warning 并以 0 状态退出， 不继续生成下载脚本。

常见问题

现象 可能原因 处理建议
报错：project id must start with HRA, CRA, or http 输入了不支持的编号，例如 XDA/XDB。 改用 HRA/CRA 编号，或直接传入可访问的完整下载 URL。
无法找到 CRA download URL browse HTML 中没有唯一的 download 链接，或网页结构变化。 使用 --force 重新下载 HTML；仍失败时检查 index.html 中链接。
md5sum.txt 下载 404 最终 URL 不含 md5sum.txt，或项目目录层级不是脚本预期。 确认 url.txt 中的 URL 是否是实际数据根目录。
没有生成 FASTQ 下载脚本 md5sum.txt 中没有以 q.gz 结尾的条目。 查看 MD5_all.txt 确认文件后缀。
Aspera 脚本无法运行 缺少 Aspera 客户端或 key 文件。 安装 Aspera Connect，并按日志提示下载 aspera01.openssh。
