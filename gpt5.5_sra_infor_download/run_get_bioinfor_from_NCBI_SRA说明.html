Python 脚本说明文档

run_get_bioinfor_from_NCBI_SRA.py

该脚本用于从 NCBI SRA 获取测序 run 的文件信息，并自动生成可直接执行的下载脚本和 MD5 校验文件。输入可以是单个 SRR / ERR / DRR run，也可以是项目或研究编号，例如 PRJNA531644、 SRP191521。

Python 3 NCBI E-utilities TSV 输出 wget / axel / aria2c / aws s3 cp MD5 校验

快速开始

脚本只依赖 Python 标准库，不需要额外安装第三方 Python 包。运行前需要保证输出目录已经存在，并且当前网络可以访问 NCBI。

terminal
# 查看帮助
python run_get_bioinfor_from_NCBI_SRA.py -h

# 单个 SRA run
python run_get_bioinfor_from_NCBI_SRA.py -i SRR8869110 -o ./download2

# BioProject 项目号
python run_get_bioinfor_from_NCBI_SRA.py -i PRJNA531644 -o ./download1

# SRA Study 编号
python run_get_bioinfor_from_NCBI_SRA.py -i SRP191521 -o ./download1
最小使用条件：准备一个已存在的输出目录，传入 -i 指定 ID，传入 -o 指定输出目录。 脚本会自动判断输入是单个 run 还是项目/研究编号。

参数与使用示例

参数 是否必需 默认值 说明 示例
-i, --input 是 无 SRA run、study 或 BioProject ID。run 号通常以 SRR、ERR、DRR 开头。 SRR8869110
-o, --output_dir 否 ./ 输出目录。脚本只检查目录是否存在且是目录，不会自动创建目录。 ./download1

输入输出文件说明

输入

输入类型 识别方式 处理路径 说明
单 run ID SRR* / ERR* / DRR* 直接调用 efetch 脚本将输入视为一个具体 run，并只生成该 run 的文件信息。
项目或研究 ID 其他字符串，例如 PRJNA*、SRP* 先 esearch，再 efetch 脚本先搜索 SRA 记录 UID，再从 XML 中解析所有 run accession。

输出

文件名模式 生成场景 内容 格式
{RUN}_infor.xls 每个 run 该 run 的 SRAFile 和 Alternatives 记录。虽然扩展名是 .xls，实际是 Tab 分隔文本。 TSV
{INPUT}_run_list.txt 项目/研究 ID 从项目/研究解析出的 run accession，每行一个。 TXT
{INPUT}_all_run_infor.xls 项目/研究 ID 项目下所有 run 的文件信息合并表。 TSV
{PREFIX}_download_wget.sh run 和项目/研究 使用 wget -c -O 下载匿名 HTTPS 链接。 Bash
{PREFIX}_download_axel.sh run 和项目/研究 使用 axel -a -n 10 -o 下载匿名 HTTPS 链接。 Bash
{PREFIX}_download_aria2c.sh run 和项目/研究 使用 aria2c -c -x 8 -j 8 -s 8 -o 下载匿名 HTTPS 链接。 Bash
{PREFIX}_download_aws_s3_cp.sh run 和项目/研究 对 s3:// 或 *.s3.amazonaws.com 格式的匿名公开链接，使用 aws s3 cp --no-sign-request 下载，并在本地文件已存在时跳过。 Bash
{PREFIX}_download_md5.txt run 和项目/研究 可下载文件的 MD5 和文件名，适配 md5sum -c。 TXT

脚本思路与流程

脚本的核心思路是：先把用户输入解析为一个或多个 run accession，再逐个抓取 XML 文件信息，最后把匿名可下载链接转换成下载命令和校验清单。

解析参数读取 -i 和 -o
→
检查输出目录确认目录存在
→
解析输入 IDrun 直通，项目先搜索
→
获取 SRA XML调用 NCBI efetch
→
解析文件记录SRAFile + Alternatives
→
写出结果TSV、脚本、MD5

单 run 输入

SRR8869110 这类输入会直接进入 fetch_sra_xml()，输出前缀就是 run ID。

项目/研究输入

PRJNA531644、SRP191521 这类输入会先通过 search_sra_uids() 找到 SRA UID，再解析出所有 run。

函数功能说明

函数 输入 输出 功能说明
parse_args()命令行参数argparse.Namespace定义并解析 -i/--input 与 -o/--output_dir。
fetch_url(url, retries, timeout)URL、重试次数、超时UTF-8 文本带 User-Agent 请求远程 URL，失败时按次数重试。
check_output_dir(output_dir)Path无检查输出路径是否存在且为目录。
fetch_sra_xml(srr)run IDXML 文本调用 NCBI EFetch 获取单个 run 的 XML。
search_sra_uids(query)项目/研究查询词UID 列表调用 NCBI ESearch 搜索 SRA 记录 UID。
fetch_sra_xml_by_uids(uids)UID 列表XML 文本用逗号拼接 UID 后批量调用 EFetch。
parse_run_accessions(xml_text)XML 文本run ID 列表从 RUN 节点中解析 SRR/ERR/DRR accession，并去重排序。
resolve_input_to_runs(input_id)用户输入 IDrun ID 列表判断输入类型，将单 run 或项目/研究统一解析为 run 列表。
append_new_keys(columns, record)列名列表、记录无当 XML 中出现新属性时追加为输出列。
parse_sra_files(xml_text, srr)XML 文本、run ID列名与行记录解析 SRAFile 及其 Alternatives，组合为表格行。
read_table(path)TSV 路径列名与行记录读取已存在的结果文件，用于跳过重复抓取后重新生成脚本。
write_table(path, columns, rows)路径、列名、记录无按 Tab 分隔写出信息表。
merge_columns(rows)所有行记录列名列表项目级汇总时合并默认列和实际出现的扩展列。
get_anonymous_download_rows(rows)行记录可下载行筛选 access_type == anonymous 且存在 url 的记录。
build_s3_url(url)下载 URLS3 URL 或空值将 s3:// 或 *.s3.amazonaws.com HTTPS 链接规范化为 s3://bucket/key 格式。
build_download_command(tool, row)工具名、行记录命令字符串根据 wget、axel、aria2c 或 aws s3 cp 生成下载命令。
write_download_scripts(output_dir, prefix, download_rows)目录、前缀、可下载行脚本路径列表分别生成多种下载工具的 Bash 脚本并设置可执行权限。
write_md5_file(output_dir, prefix, download_rows)目录、前缀、可下载行MD5 文件路径写出 md5 filename 格式的校验文件。
process_run(run_id, output_dir)run ID、输出目录该 run 行记录处理单个 run；若信息表已存在，则复用旧表并重建脚本与 MD5。
write_run_list(output_dir, input_id, runs)目录、输入 ID、run 列表列表文件路径写出项目/研究解析得到的 run 列表。
write_project_outputs(output_dir, input_id, all_rows)目录、输入 ID、所有行无生成项目级合并信息表、下载脚本和 MD5 文件。
main()无退出码组织完整流程，成功返回 0，失败返回 1。

注意事项

输出目录不会自动创建。如果 -o 指向不存在的路径，脚本会报错并返回 1。先使用 mkdir -p download1 或手动创建目录。
  • 信息表扩展名为 .xls，但本质是 UTF-8 TSV 文本；可用 Excel、WPS、LibreOffice 或文本编辑器打开。
  • 脚本只把 access_type 为 anonymous 且存在 url 的记录写入下载脚本；AWS S3 下载脚本还要求 URL 是 s3:// 或 *.s3.amazonaws.com 格式。
  • 某些 URL 是 s3:// 或需要云身份访问，这类记录通常不会出现在匿名下载脚本中。
  • 如果某个 {RUN}_infor.xls 已存在，脚本会跳过重新抓取该 run，并基于现有表重新生成下载脚本和 MD5 文件。
  • 下载脚本包含 set -euo pipefail，任一下载命令失败时脚本会停止。
  • 项目/研究编号可能解析出大量 run，运行时间和 NCBI 请求次数会明显增加。
  • 下载体积可能很大，执行下载脚本前应确认磁盘空间、网络稳定性和目标目录。
  • Windows PowerShell 可以运行 Python 脚本；生成的 .sh 下载脚本建议在 Git Bash、WSL 或 Linux/macOS 环境执行。

常见问题

报错：Output directory does not exist

输出目录不存在。先创建目录，再重新执行脚本。

fix.sh
mkdir -p download2
python run_get_bioinfor_from_NCBI_SRA.py -i SRR8869110 -o ./download2

报错：No SRA records found

输入 ID 可能写错，或 NCBI SRA 中没有该记录。检查编号前缀和完整数字。

下载脚本为空或命令很少

脚本只导出匿名可访问链接。如果 XML 中主要是云身份或受限访问链接，下载脚本会相应减少。

MD5 校验失败

优先检查下载是否中断、文件名是否被改动、是否有旧文件残留。可删除失败文件后重新执行下载脚本。
